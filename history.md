> 대상: `auto_synthesize_gui_dpv2.py`
> 다중 엑셀 일괄 마스킹 + 파일 간 컬럼 관계 보존 + 클로드코드용 테이블 생성 작업서(.md) 자동 생성
> (삼성중공업 생산 DT센터)

이 문서는 한 세션 동안 진행한 기능 추가·버그 수정·성능 개선 내역을 정리한 것입니다.

## 1. AI 채팅 복붙 도우미 (컬럼 설명 입력 자동화)

외부 AI(Claude/ChatGPT 등) 채팅창에 **사람이 직접 복붙**하는 흐름을 돕는 기능. (자동 API 연동 아님)

- **📋 컬럼+질문 프롬프트 복사**: 이 파일의 컬럼 목록 + "컬럼별 설명을 `컬럼명: 설명` 형식으로 답해줘"라는 **질문 프롬프트까지 한 번에** 클립보드에 복사. AI 채팅창에 그대로 붙여넣으면 됨.
- **📥 AI 답변 설명 일괄 붙여넣기**: AI 답변을 붙여넣으면 각 컬럼 설명란을 **한 번에** 채움. 한 줄씩 입력할 필요 없음.
  - 매칭 방식 자동 감지: **이름 기반**(`컬럼명: 설명`) / **순서 기반**(한 줄에 설명 1개)
  - 항목 구분: **줄바꿈 또는 백슬래시(`\`)** 모두 인식 → 줄바꿈을 못 하는 사내 AI 대응
  - 선행 번호/불릿(`1.`, `2)`, `-`) 자동 제거

관련 메서드: `_build_ai_prompt`, `_copy_columns_csv`, `_parse_description_block`, `_apply_descriptions`, `_paste_descriptions_dialog`

## 2. 단일 파일(1개)도 실행 가능

파일이 1개일 때 `전체 합성 데이터 생성 실행` 버튼이 활성화되지 않던 문제 수정.

- 파일 간 관계는 2개 이상일 때만 존재하므로, **1개 파일은 관계 확정 게이트를 면제**.
- `_update_run_state`: `rel_gate = relationships_confirmed or len(files) <= 1`
- `_run`: `len(files) > 1`일 때만 관계 확정 요구

## 3. 샘플 테스트 데이터 생성기 (`make_sample_data.py`)

앱 기능 검증용 관계 얽힌 엑셀 4종 생성.

| 파일 | 내용 | 테스트 포인트 |
|---|---|---|
| `1_직원정보.xlsx` | 사번·이름·부서·입사일·연락처·이메일·기본급 | ID/PII/날짜/null |
| `2_프로젝트.xlsx` | 프로젝트코드·발주사·PM담당자·시작일·종료일·계약금액 | 시작<종료 제약 |
| `3_작업배정.xlsx` | 작업ID·사번·프로젝트코드·진행률·투입공수 | 관계 키 2개 |
| `4_호선정보.xlsx` | 호선번호·프로젝트코드·선종·선주·전장·재화중량 | 관계 키 |

관계 설계: **사번**(직원↔작업), **프로젝트코드**(프로젝트↔작업↔호선, 3파일 그룹), 회사명(발주사↔선주), 이름(PM담당자↔직원).

## 4. 반응형(Responsive) 레이아웃 — 작은 노트북 화면 대응

13~15인치(예: 1536×864, 사용가능 높이 793px)에서 잘림/안보임 다수 → 웹 프론트엔드식 반응형으로 개선.

- **창 크기**: 화면 크기에 맞춰 자동 산정(`min(1280, 화면-여백)`) + 중앙 배치, 최소 크기 축소(이전 1200×980 → 잘림 유발).
- **파일 탭 버튼(FileTabBar)**: 한 줄 고정 → **flex-wrap 식 자동 줄바꿈**(`_reflow`, `<Configure>` 실시간 재배치).
- **파일별 테이블명 입력란**: 한 줄 6개씩, 초과 시 자동 줄바꿈(grid).
- **세로 좁은 화면**: 실행 로그(7→3줄)·파일 목록(4→3줄) 높이 축소 — **가로폭이 아닌 창 높이 기준**으로 판단.
- **단계 인디케이터**: 긴 힌트를 아래 줄로 분리.
- **확정 버튼류**: 스크롤 영역 밖 고정 — ④ 파일 확정 버튼은 탭 상단, ③ 관계 확정 버튼은 탭 하단(`side=BOTTOM`)에 고정해 화면 크기와 무관하게 노출.

## 5. 성능 개선 — 30,000행+ / 다중 파일 렉 해결

### A. 전체 분석을 백그라운드 스레드로
`📊 전체 파일 분석`이 UI 스레드에서 동기 실행되어 창이 얼어붙던 문제 → **워커 스레드**로 분리.
- `_analyze_worker`(백그라운드, Tkinter 미접근) / `_analyze_progress` / `_analyze_finish`(메인 스레드 UI 갱신)
- 진행률 실시간 표시, 분석 중 버튼 비활성·중복 실행 방지

### B. 무거운 연산 최적화
- `compute_value_overlap`: 파이썬 루프 → **pandas 벡터화**, 고유값 5만 초과 시 **해시 기반 일관 샘플링**(MinHash식, 겹침 보존)
- `_is_id_col`, `auto_detect_column_type`: 전체 행 → **앞 1000개 샘플** 정규식 판별
- `_postprocess_dataframe`: 날짜 아닌 텍스트에 전체 `to_datetime` 시도하던 것 → **50개 샘플 선검사 후** 변환

## 6. 따로 올릴 때 vs 같이 올릴 때 (개념 정리)

관계 있는 테이블을 **따로(개별 세션) 올리면**:
- **복구(원본 복원)**: 정상 — 파일별 `_변환키.json`으로 테이블 단위 복원
- **관계 기반 바이브코딩**: ❌ — 공유 키(사번 등)가 테이블마다 다른 가짜값 → 마스킹 DB에서 JOIN 불일치, `관계메타.json` 미생성

→ **관계 있는 테이블은 반드시 한 번에 같이 올려서** 합성해야 공유 키가 같은 가짜값을 공유하고 관계가 보존됨.

`tableanddata.py`: 합성 데이터를 DB에 적재(CREATE TABLE → INSERT → PK/FK/INDEX)하는 실행 스크립트. 마스킹 컬럼명+합성 데이터만 포함(원본 PII 미포함).

## 7. 메일 발송 (`send_mail.py`)

작업 결과물 파일을 메일로 보내는 요청 대응.

- 연결된 Gmail MCP는 **draft(임시저장)만** 지원 → 실제 발송 불가
- 대안으로 **smtplib + Gmail 앱 비밀번호** 방식의 `send_mail.py` 작성
  - `.env`에 `GMAIL_USER` / `GMAIL_APP_PW`(16자 앱 비밀번호) 설정(1회)
  - `.py` 등 차단되기 쉬운 확장자는 자동 zip 첨부
  - `python send_mail.py <받는주소> <파일> [--subject ...] [--body ...]`
- 주의: 내부망에서 `smtp.gmail.com:465` 아웃바운드가 막혀 있으면 사내 SMTP relay 필요
