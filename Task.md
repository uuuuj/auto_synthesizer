## ✅ 이슈 1: 📋 "컬럼+질문 프롬프트 복사" 버튼 — 멈춤(hang) (해결, 사용자 검증 대기)

### 증상 (사용자 환경)
- 버튼을 눌러도 **완료 팝업이 안 뜸** / **클립보드에 복사도 안 됨**
- 창이 뒤에 가려진 것 **아님** (Alt+Tab으로 확인) — 누르면 앱이 멈추는 것처럼 보임

### 확정된 원인
- 버튼 클릭 시 **메인 스레드에서 곧바로 Tk 클립보드(`clipboard_clear/append`)를 동기 호출** →
  사내 보안/DLP가 클립보드 접근을 가로채면 그 줄에서 **앱 전체가 행(hang)**.
  → 예외가 아닌 '멈춤'이라 기존 `try/except`(TclError)로는 못 막았고,
    팝업(`showinfo`)까지 도달하지 못해 "복사도 안 되고 팝업도 없음"과 정확히 일치.
- ※ 복사 대상은 **컬럼 이름 + 고정 질문 문구뿐**(데이터 행 아님)이라 **파일 크기와는 무관**.

### 적용한 수정 (`auto_synthesize_gui_dpv2.py`)
- `_copy_columns_csv` 재작성 — 메인 스레드에서 동기 Tk 클립보드 호출 제거:
  1. **선택 가능한 텍스트 창**(`_show_copyable_prompt`)을 먼저 띄움 — 클립보드 호출 0 → 안 멈춤.
     텍스트 전체 선택 + 포커스 상태라 `Ctrl+C`로 즉시 복사 가능. 위젯 빌드는 try/except로 감쌈.
  2. 자동 복사는 **백그라운드 스레드 + `clip.exe`(timeout=3s)**(`_clipboard_set_bg`) — 막혀도 UI 안 얼음.
     `locale.getpreferredencoding()`(한글 Windows=cp949)로 인코딩, `root.after`로 결과 보고.
  3. 모달 `showinfo` 제거(추가 블로킹 지점 제거). 창의 [📋 다시 복사] 버튼도 동일 백그라운드 경로 사용.
- 검증: 개발 PC에서 `clip.exe + cp949` 한글 복사 round-trip OK(Tkinter `clipboard_get`로 확인).

### 남은 확인
- [ ] **사용자 PC(DLP 환경)에서 실제로 안 멈추는지** 최종 확인 — 멈추면 창은 떠야 정상.
- [ ] DLP가 `clip.exe`까지 막을 경우: 창에서 `Ctrl+C` 수동 복사가 동작하는지 확인.

## ✅ 이슈 3: 파일 탭 버튼 / "이 파일 확정" 버튼이 안 보임 (해결, 사용자 검증 대기)

### 확정된 원인 — 좁은 화면에서 노트북(④ 컬럼 편집) 영역 붕괴
- 메인 레이아웃에서 **노트북만 `expand=True`**라, 창이 짧아지면 다른 섹션은 자연 높이를
  유지하고 **노트북만 찌부러진다.** 창 높이가 임계 아래로 내려가면 파일 탭 본문(body)이
  몇 px로 붕괴 → 그 안의 '이 파일 확정' 버튼이 아예 화면에 안 그려짐(`mapped=False`).
- **DPI 가상화가 트리거**: 이 앱은 DPI 미인식 → Windows가 화면을 배율만큼 축소해 알려줌.
  13인치 고배율(150~175%)이면 앱이 보는 화면 높이가 ~576~720px → 창 높이가 560~620으로
  만들어져 임계 아래로 떨어짐. (개발 PC는 1920×1080@125% → 앱은 1536×864로 인식 → 창 764
  → 안 붕괴되어 그동안 재현이 안 됐던 것.)
- ※ DPI 인식을 켜는 건 **해결책 아님** — Tk가 폰트 스케일도 같이 키워(1.33→1.67) 들어가는
  내용량은 그대로. 실측으로 확인함.

### 측정한 임계 (테스트 하네스로 확인)
- 수정 전: 창 높이 ≤ 600px → 확정 버튼 붕괴(안 보임). ≥ 620px만 정상.
- 수정 후: **창 높이 560px(최저값)까지 전부 정상 표시** (모든 화면 크기 커버).

### 적용한 수정 (좁은 화면일수록 다른 섹션을 압축해 노트북 최소 높이 확보)
- 분석요약(파일탭 내부) 줄수: `_win_h<700` 시 1줄, 파일 리스트 2줄, 로그 `<640` 시 1줄.
- 섹션 패딩 `_win_h<700` 시 2px, 단계바 힌트 줄은 `<660` 시 숨김(위젯은 유지해 콜백 안전).
- 검증: 샘플 4파일 로드→관계확정→④탭까지 자동 구동 후 확정버튼 `winfo_ismapped` 확인.

### 남은 확인
- [ ] 사용자 13인치 PC에서 파일 탭 버튼·확정 버튼이 실제로 보이는지 최종 확인.
      (참고용: 사용자 PC의 **화면 해상도 + 디스플레이 배율%**를 알려주면 정확히 맞출 수 있음)
